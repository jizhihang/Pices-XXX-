17  code will be internal to BaseLibrary (KKU) classes. For the most part all functions that

23  \section OtherRequiredLibraries Other Required Libraries
24  To use al the classes in this Library you are also going to need the outside libraries fftw
25  and zlib123. "fftw" stands for "Fastest Fourier Transform in The West". It can be
26  down loaded from http://www.fftw.org/. "zlib123" is a library that is used to compress and
27  uncompress data. can be found at http://www.zlib.net/. It is used by the Compressor class.

31  \section KKQueue KKQueue
32  I have a container template called KKQueue that is derived from the std::vector<> template.
33  It is specifically meant to work with pointers to its contents and understands the
34  concept of ownership. That is a instance of a KKQueue template either owns its contents
35  or it does not. If it owns them it will call the individual destructors for each item that it
36  contains when its destructor is called. Many classes that I have written use this template so
37  it would be worth reading its documentation.
38  \n\n

41  \section ImageProcessing Image Processing
42  There are several classes that aid in Image Processing. The primary class is Raster which
43  can handle both Grayscale and Color images. It supports various morphological operations
44  as well as Connected Component Analysis, Fourier transform, etc. Other classes that work
45  with Raster are Blob, BMPheader, BMPheader.h, ContourFollower, Histogram, ImageIO.h,
