4 /**
5  *@class MLL::MLClass
6  *@brief Represents a "Class" in the Machine Learnining Sense.
7  *@author Kurt Kramer
8  *@details
9  *@code
10  **********************************************************************
11  ** MLClass *
12  ** *
13  ** Each instance of this class represents a single Class as used in *
14  ** Machine Learning sense. Each instance of 'FeatureVector' class *
15  ** will point to an instance of this class. There can only be one *
16  ** instance of each Class in memory. Specifically the 'name' field *
17  ** will be unique. This is enforced by making the constructor and *
18  ** destructor private. The only way to create a new instance of a *
19  ** 'MLClass' object is to call one of the the static methods of *
20  ** 'CreateNewMLClass'. These methods will look for a instance *
21  ** of 'MLClass' that already exists. If one does not they will *
22  ** then create a new one. *
23  ** *
24  ** Please refer to MLClassList at bottom of file. That class is *
25  ** the object you will most be using when dealing with Images. *
26  ** classes. *
27  ** *
28  ** PicesClass
29  ** There is a special relationship between this class and a class *
30  ** called 'PicesClass' in the 'PicesInterface' library'. PicesClass*
31  ** is a managed c++ version of this class. There is a one-to-one *
32  ** correspondence between the two classes. When ever a instance *
33  ** of 'PicesClass' gets created it will automatically call the *
34  ** the static method 'MLClass::CreateNewMLClass' to get the *
35  ** unmanaged version of the class. *
36  ** *
37  ** *
38  ** classId - A unique number that is assigned by the Pices *
39  ** DataBase when a new class is added to it. *
40  ** *
41  ** name - Name of class; This name will be unique for the *
42  ** process. Underscore characters have special meaning *
43  ** to several methods in PicesLibrary. They help *
44  ** separate the name into multiple class levels. *
45  ** *
46  ** num - Used by SVMModel when creating Training Models and *
47  ** Classifying an instance of a imageFeatures *
48  ** *
49  ** parent - Pointer to a instance of 'MLClass' that is the *
50  ** parent to this instance. And example would be a *
51  ** class called 'Copepod_Calanoid' will have a parent *
52  ** called 'Copepod'. The parent class to all classes *
53  ** is "AllClasses" *
54  ** *
55  ** unDefined - When set to true indicates that this class is not *
56  ** a Plankton or has not yet been classified as a *
57  ** Plankton. This is set automatically when name is *
58  ** set. Names such as NOISE, UNKNOWN, UNDEFINED, NOISY, *
59  ** and NONPLANKTON will be set to true otherwise *
60  ** false. The names are NOT case sensitive so for *
61  ** example Noise, UnKnown, and noisy would set it *
62  ** false. *
63  **********************************************************************
64  *@endcode
65  *@see MLL::MLClassList, MLL::FeatureVector, PicesInterface::PicesClass
66  *@see MLL::DataBase::MLClassLoad, MLL::DataBase::ImageInsert
67  */
