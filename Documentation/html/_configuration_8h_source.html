14  /**
15  *@brief General purpose Configuration File manager class.
16  *@details This class will read and write configuration files. It understands the concept of Logical Sections and
17  * Variables. You will be able to programmatically define Sections and Settings in these sections. Each Setting
18  * will have a related value stored with it.
19  *
20  *@code
21  *Example Configuration File
22  *===========================================================
23  * [Header]
24  * ClassifierType = SVM
25  * Parametrs = -K RBF -S SVC -G 1.76 -C 12
26  * RootDir = ${PicesHmeDir}\TraningLibraries\SimpleLibrary\
27  *
28  * [Class]
29  * Name = Copepod
30  *
31  * [Class]
32  * Name = Larvacean
33  *===========================================================
34  *@endcode
35  *
36  * There are three sections defined in the example above; Header, Class, and Class. Note
37  * that Section names do not have to be unique. You can access sections by name or index.
38  * If the name is not unique the first instance will be returned. Index values start at
39  * the top of the file, that is the first section to appear in the file is section index
40  * 0. The Setting names are of the format "SettingName = Value". Again you will be able
41  * to access Setting by either name or index.
42  */
